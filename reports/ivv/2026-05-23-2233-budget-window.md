## Why this run matters (the measurable payoff)

The independent suite **killed 14 surviving mutants + 1 uncovered mutant that the
Designer's own tests missed** — concentrated exactly on the two behaviors the
architect had just specified (the R9 tie-break comparator and the INV5 clamp).
This is direct, quantified evidence that fresh-context independent verification
detects gaps that self-authored tests do not — the question that motivated
building this flow.

It also surfaced **three genuine spec ambiguities** in wording I (the Designer)
had written, none of which were logic bugs, all of which the architect resolved —
turning latent under-specification into pinned, tested behavior.

## Honest residual (what independence could NOT cover)

- **Shared-spec blind spot** (Knight & Leveson): the reference model and the DUT
  both descend from the same contract, so a misreading common to both would pass
  the scoreboard. Mitigated — not eliminated — by oracle-free checks (hand-derived
  golden labels/dates incl. leap-day and year-roll; metamorphic relations
  amount-blind / date-translation / permutation-invariance; structural conservation).
- Error contract verified only to the level of "throws + message references the
  field," not exact error type/string.
- **Tooling limitation found:** Stryker cannot run against a custom Vitest config
  (its sandbox fails to resolve the project's `.svelte-kit/tsconfig.json`
  `extends` path). The VE measured combined kill power by temporarily adding the
  IV&V file to `vitest.config.ts` include, then reverting (net-zero diff verified).
  The reference-model self-mutation check independently confirms scoreboard teeth.
  → captured in the verification-engineer charter for future runs.
